# A speed limit on deception in robot swarms — code and data

[![DOI](https://zenodo.org/badge/1307548050.svg)](https://doi.org/10.5281/zenodo.21471557)

Simulation code, flight-plan packages, and flight data for the manuscript

> L. Shi, Z. Ma, P. Zhu, *A speed limit on deception in robot swarms*,
> submitted to PNAS (2026).

Every quantitative claim in the paper (all figures, Table 1, and the
numbers quoted in the text) is reproduced by the scripts in this
repository. All simulations use fixed random seeds, so every run
reproduces the published numbers exactly.

## Environment

Python 3.9 or later with

```
pip install -r requirements.txt
```

The only dependencies are `numpy` and `matplotlib`. Figure fonts
(Latin Modern, matching the manuscript body font) are vendored in
`fonts/` and load automatically through `nc_style.py`. No system font
installation is needed.

## Reproducing the figures and tables

Run each script from the repository root. Outputs are written to the
working directory.

| Item in paper | Script | Output | Runtime |
|---|---|---|---|
| Fig. 1 (concept) | `python make_fig_concept.py` | `fig_concept.pdf` | seconds |
| Fig. 2 (N=2000 expulsion) | `python make_fig_traj_large.py` | `fig_traj_large.pdf` | ~2 min |
| Fig. 3 (five-controller baseline), Fig. S1 (scaling), Fig. S3 (adaptive gray zone) | `python make_figs_nc.py` | `fig_baseline.pdf`, `fig_scaling.pdf`, `fig_adaptive.pdf`, `fig_nc_numbers.txt` | ~10 min |
| Fig. 4 (walk-off cliff) | `python make_fig_gps_nc.py` | `fig_gps.pdf` | ~1 min |
| Fig. 5 (hardware composite) | `python make_fig_hw_nc.py` | `fig_hw_nc.pdf` | seconds |
| Figs. S2 (trust regimes), S4 (noise sweep), S5 (unicycle) | `python make_figs_si_nc.py` | `fig_trust.pdf`, `fig_noise.pdf`, `fig_nl.pdf` | ~5 min |
| Table 1, flight plans | `python gen_hw_traj_sweep.py` | `hardware_plan_sweep/` | ~5 min |

`fig_nc_numbers.txt` collects the numbers quoted in the manuscript text
(baseline disagreement levels, walk-off boundary, peak accepted
displacement, and so on).

All physical constants are stated in the SI Appendix, Table S3, and are
set at the top of each script. The reachable-tube radius used
throughout is `RHO = U*(exp(L*DELTA)-1)/L`, the radius about the
nominally propagated anchor (SI Appendix, Note 2).

## Flight experiments

`hardware_plan_sweep/liars1` … `liars4` are the four flight-plan
packages flown on the Crazyflie 2.1+ / Nokov motion-capture arena, one
per adversary-count configuration (1–4 liars out of 10). Each package
contains

- `uav01.csv` … `uav10.csv`: per-vehicle waypoint files at 20 Hz
  (vehicle number = landing-pad number),
- `ztswarm_traj.csv` and `manifest.csv`: the full closed-loop
  trajectory and the package manifest,
- `ztswarm_overview.png`, `ztswarm_traj.gif`, `1.png` … `4.png`:
  previews of the planned flight,
- `README.txt`: flight instructions.

The plans are generated by `gen_hw_traj_sweep.py` from the same trust
law, reachable-set verification, and safety program as the
simulations, at the platform's physical scale (U = 0.6 m/s, 50 Hz,
20 cm safety margin). The per-row statistics of Table 1 are computed
by this closed-loop generator over four seeds and five migration
routes per configuration.

The deposited flight record comprises the flown plan packages above
and the motion-capture top views of the four flown configurations
(`fig_hw_flight.png`, shown as Fig. 5c–f of the main text), together
with the testbed photograph (`fig_platform.jpg`, Fig. 5a).
